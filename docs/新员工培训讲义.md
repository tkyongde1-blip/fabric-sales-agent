# 新员工培训讲义
## 面料销售助手使用培训

版本：最终稳定商用版  
适用对象：新入职销售、客服、运营同事

---

# 一、系统是做什么的

面料销售助手是一套微信自动值守系统，主要负责：

- 自动扫描微信未读消息  
- 识别客户常见需求  
- 生成标准回复  
- 自动发送  
- 记录审计日志  
- 在需要时让人工接管

它不是来替代销售，而是把重复劳动接过去，让销售把时间留给真正需要判断力的客户。

---

# 二、系统能处理哪些消息

## 1. 适合自动处理

- 询价  
- 规格确认  
- 样品咨询  
- 发货时间  
- 付款方式  
- 常规下单意向

## 2. 不适合完全自动处理

- 强烈压价  
- 投诉与售后  
- 质量争议  
- 大客户长期合作  
- 模糊需求  
- 需要拍板的特殊政策

这类场景应使用“人工接管”。

---

# 三、日常工作流程

## 1. 开班流程

1. 打开微信  
2. 打开助手程序  
3. 确认微信已连接  
4. 确认自动处理已开启  
5. 查看“今日汇总”

## 2. 值守流程

- 普通客户：让系统自动回复  
- 重点客户：加入白名单并重点关注  
- 复杂客户：切换人工接管  
- 不再需要自动回复的客户：移出白名单或暂停  
- 不合作客户：拉黑

## 3. 收班流程

1. 查看当日汇总  
2. 查看失败告警  
3. 检查是否有忘记释放的人工作业  
4. 记录次日跟进名单

---

# 四、核心按钮说明

## 加入白名单

用途：允许系统自动回复该客户。  
适合：明确需要持续跟进的客户。

## 移出白名单

用途：不再让系统自动回复该客户。  
适合：暂时观察或低优先级客户。

## 拉黑当前客户

用途：永久停止自动回复。  
适合：恶意骚扰、不再合作、明确无需继续跟进的客户。

## 暂停自动回复

用途：短期停用。  
适合：客户正在被人工处理，但之后还可能恢复自动值守。

## 人工接管

用途：系统完全让位给人。  
适合：谈判、投诉、大客户、复杂需求。

## 释放接管

用途：把客户重新交回系统自动值守。  
适合：人工阶段结束后。

# 五、系统的安全机制

## 1. 群聊过滤

系统会自动跳过群聊，避免误发。

## 2. 黑名单过滤

黑名单客户不会被自动回复。

## 3. 白名单控制

如果配置了白名单，系统只对名单内客户自动回复。

## 4. 重复发送抑制

同一条消息不会重复发送。

## 5. 发送失败重试

失败后会自动重试，并记录告警。

## 6. 上下文继承

同一客户后续追问时，系统会继承最近一次有效产品信息，不会轻易“失忆”。

# 六、常见问题

## Q1：客户说“能便宜点吗”，系统怎么处理？

如果前面已经聊过具体产品，系统会继承上下文继续回复。  
如果这是关键谈判，建议人工接管。

## Q2：为什么有些客户没有自动回复？

先看四件事：
1. 是否白名单外  
2. 是否被暂停  
3. 是否被人工接管  
4. 是否群聊

## Q3：系统回复错了怎么办？

1. 先暂停该客户自动回复  
2. 人工接管  
3. 记录样本  
4. 后续纳入规则优化

## Q4：哪里看今天运行情况？

- 点击“今日汇总”  
- 或查看 `data\daily_summaries\`

# 七、异常处理训练

## 场景 1：客户没有收到自动回复

处理顺序：
1. 查看状态栏  
2. 看是否在白名单  
3. 检查是否被暂停/接管  
4. 手动读取当前消息  
5. 必要时手动回复

## 场景 2：发送失败

处理顺序：
1. 查看失败告警日志  
2. 检查微信窗口是否正常  
3. 重启微信和助手  
4. 必要时手动补发

## 场景 3：重点客户开始压价

处理顺序：
1. 立即人工接管  
2. 不让系统继续插话  
3. 由销售本人完成成交谈判

# 八、培训考核清单

新人需独立完成以下动作：

- [ ] 启动程序并确认连接  
- [ ] 加入一个客户到白名单  
- [ ] 暂停一个客户自动回复  
- [ ] 人工接管一个客户  
- [ ] 释放接管  
- [ ] 查看今日汇总  
- [ ] 找到失败告警日志  
- [ ] 说清楚何时必须人工接管

# 九、最后要记住的话

系统负责稳定执行，  
人负责判断关系。

真正优秀的使用者，不是让系统说得最多的人，  
而是知道什么时候该让系统沉默的人。
