## 上线检查清单 + 日常运营 SOP + 异常处理手册

版本：最终稳定商用版

# 一、上线检查清单

## 1. 环境准备

- [ ] 微信 PC 端已安装、已登录、能正常收发消息  
- [ ] 运行电脑已关闭自动休眠，屏幕不会在值守期间锁定  
- [ ] 微信主窗口可见，未被最小化到托盘  
- [ ] 助手程序位于固定目录，建议使用：
  - `E:\projects\fabric-sales-agent\dist\fabric_sales_agent.exe`
- [ ] `data\fabric_sales.db` 数据库文件存在且可读写  
- [ ] 当天网络稳定，微信登录状态正常  

## 2. 程序启动检查

- [ ] 启动助手后，状态栏显示微信连接成功  
- [ ] 自动模式已开启  
- [ ] 当前聊天窗口可被读取  
- [ ] 白名单、黑名单、暂停、人工接管按钮可正常使用  
- [ ] “今日汇总”按钮可以生成内容  

## 3. 自动化链路检查

- [ ] 新客户消息到达后，助手能识别未读会话  
- [ ] 能打开未读聊天并读取最后一条客户消息  
- [ ] 单聊会自动回复  
- [ ] 群聊会自动跳过  
- [ ] 黑名单客户会自动跳过  
- [ ] 被暂停或人工接管的客户不会被自动回复  
- [ ] 回复成功后，能写入审计记录  
- [ ] 发送失败时，能重试并写入告警日志  

## 4. 上线前强制验证

- [ ] 用 1 条已知客户消息做人工试发  
- [ ] 用 1 条未读真实消息验证自动回复  
- [ ] 用 1 条重复消息验证不会重复发送  
- [ ] 用 1 个群聊验证系统会跳过  
- [ ] 用 1 个白名单/黑名单客户验证策略生效  
- [ ] 检查以下文件是否正常生成：
  - `data\auto_reply_alerts.log`
  - `data\daily_summaries\YYYY-MM-DD.txt`

## 5. 上线放行标准

满足以下条件才允许正式值守：

1. 自动读取成功  
2. 自动回复成功  
3. 群聊过滤成功  
4. 黑名单过滤成功  
5. 重复消息不重复发送  
6. 失败重试可恢复  
7. 日报与审计日志正常落盘

# 二、日常运营 SOP

## 1. 每日开班流程

1. 打开微信并确认已登录  
2. 打开助手程序  
3. 确认助手已自动连接微信  
4. 查看昨日或今日日报  
5. 检查告警日志是否存在未处理失败  
6. 确认今日需要重点跟进的客户是否已加入白名单  

## 2. 客户处理原则

### 自动回复适用

- 常规询价  
- 明确产品规格咨询  
- 重复性高的样品、报价、付款、发货问题  

### 建议切换人工接管

- 客户表达强烈异议  
- 价格谈判进入关键阶段  
- 涉及投诉、退货、质量争议  
- 重点大客户  
- 客户说法含糊、需要业务判断  

## 3. 白名单管理

### 默认策略

- 如果尚未配置白名单，系统默认允许所有非黑名单单聊自动回复  
- 一旦配置了白名单，系统只会自动回复白名单联系人  

### 操作方法

- 需要自动值守的客户：点击“加入白名单”  
- 暂时不需要自动回复的客户：点击“移出白名单”  
- 永不自动回复的客户：点击“拉黑当前客户”  

## 4. 单客户控制

### 暂停自动回复

适用于：
- 客户正在等待人工跟进  
- 当前会话进入复杂谈判  
- 需要暂时观察但不想永久拉黑  

### 人工接管

适用于：
- 销售本人已经介入  
- 需要连续多轮人工对话  
- 系统继续插话会影响成交  

### 恢复自动回复

适用于：
- 人工处理结束  
- 客户重新回到标准化沟通阶段  

## 5. 每日收班流程

1. 查看“今日汇总”  
2. 复核失败发送记录  
3. 查看审计记录中的异常跳过项  
4. 清理不再需要的暂停或人工接管状态  
5. 记录当天需要次日继续跟进的客户

# 三、异常处理手册

## 1. 助手未自动回复

### 可能原因

- 微信窗口不可见  
- 当前客户不在白名单  
- 客户被暂停或人工接管  
- 客户在黑名单中  
- 消息来自群聊  
- 新消息未被识别为客户消息  

### 处理步骤

1. 确认微信窗口已打开且可见  
2. 查看客户是否在白名单  
3. 查看客户是否被暂停或人工接管  
4. 确认该聊天不是群聊  
5. 手动点击“读取当前消息”验证读取是否正常  

## 2. 回复失败

### 现象

- 状态栏显示自动回复失败  
- `auto_reply_alerts.log` 中出现记录  

### 系统已做动作

- 自动重试最多 2 次  
- 写入失败审计  
- 写入失败告警日志  

### 人工处理

1. 打开 `data\auto_reply_alerts.log`  
2. 检查失败客户与失败原因  
3. 确认微信窗口是否被遮挡、切换或异常退出  
4. 必要时手动发送一次  
5. 若连续失败，重启微信和助手后复测  

## 3. 重复发送风险

### 系统保护

- 按“联系人 + 消息内容”生成签名去重  
- 同一条消息重复出现时不会重复发送  

### 如仍发现重复

1. 立即对该客户执行“暂停自动回复”  
2. 检查聊天内容是否存在极小差异  
3. 记录样本，纳入后续回归测试  

## 4. 误回复风险

### 高风险场景

- 群聊  
- 同名联系人  
- 会话中混杂多人发言  
- 客户后续追问依赖前文上下文  

### 当前保护

- 群聊自动跳过  
- 未读扫描完成后自动回到原聊天  
- 按联系人保留最近一次有效产品上下文  
- 白名单可缩小自动回复范围  

### 处理原则

- 宁可少回，不可错回  
- 对重点客户优先人工接管  

## 5. 日报未生成

### 检查项

- `data\daily_summaries\` 目录是否存在  
- 当天助手是否正常启动过  
- 是否发生磁盘权限或写入异常  

### 处理步骤

1. 点击“今日汇总”  
2. 检查输出内容  
3. 若输出正常但文件缺失，检查目录写权限  

## 6. 数据库异常

### 现象

- 程序启动失败  
- 客户状态无法保存  
- 审计记录无法写入  

### 处理步骤

1. 备份 `data\fabric_sales.db`  
2. 关闭助手程序  
3. 确认数据库未被其他程序长期占用  
4. 重启助手  
5. 如仍异常，再进行数据库修复或回滚

# 四、运营建议

## 1. 建议的运行策略

- 新客户开发期：白名单开放一些  
- 成交关键期：重点客户人工接管  
- 售后与投诉期：全部人工接管  
- 大促期间：每日复核审计与失败日志  

## 2. 建议的监控指标

- 自动回复成功率  
- 失败发送数  
- 群聊跳过数  
- 黑名单跳过数  
- 人工接管客户数  
- 重复发送抑制数  

## 3. 建议的复盘节奏

- 每日：查看日报  
- 每周：抽查审计日志  
- 每月：复盘误判样本并更新规则

# 六、最终原则

这套系统的第一目标不是“尽量多说话”，而是：

1. 该回时稳稳地回  
2. 不该回时安静地停  
3. 出错时留下证据  
4. 复杂时把控制权交还给人  

这四条守住，系统才适合真正走进业务现场。
