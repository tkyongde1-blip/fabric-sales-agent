# 大利纺织 销售单生成规则

## 使用命令

```
cd E:/projects/fabric-sales-agent
node scripts/pipeline.js "图片路径" --customer 客户名 --name 品名 --spec 规格 --no 单号 --color 颜色
```

### 单产品示例

```
node scripts/pipeline.js "图片.png" --customer 恒泽 --name 牛奶丝双磨 --spec 190CM180G --no 12 --color 白色
```

### 双产品示例（同品名不同颜色/价格）

```
node scripts/pipeline.js "图片.png" --customer 巫石 --name 牛奶丝双磨 --spec 185CM180G --no 13 --color 黑色 --price 12 --split 20 --color2 白色 --price2 11
```

> `--split N` = 前N匹为第一个颜色/价格，后面匹数为第二个
> `--price` 不传则自动从图片总结行计算单价

## 图片解析规则

1. **全页OCR** — 先4倍放大图片，用 `chi_sim` 语言读整页，提取总结行：
   - 总匹数（340匹）
   - 总公斤（10526.5公斤）
   - 总金额（111580.90元）
   - 单价 = 总金额 ÷ 总公斤（自动计算）

2. **逐格OCR** — 按固定坐标切每个格子（400×160px），Otsu二值化后用 `eng` 数字模式读

3. **噪音过滤** — 每行不足5个15-60kg值的直接丢弃（表尾空白行）

4. **规格逐格OCR** — 幅宽/克重也用同样方式从第一行取对应格子读取，不依赖全页OCR文本
   - 幅宽格子坐标：`x=215, y=192`（标准模板），用 `eng` 数字模式
   - 克重格子坐标：`x=275, y=192`（标准模板），用 `eng` 数字模式
   - 品名格子坐标：`x=51, y=192, w=115`（标准模板），用 `chi_sim` 单行模式
   - 备选：逐格读不到时回退到全页文本搜索

5. **总公斤修正** — 逐格之和如果与图片总结行不一致，按比例缩放到总结行公斤数

---

## 销售单规则

1. **品名/规格** — 只在每组第一行写，后续行留空（合并效果）
2. **颜色/单位/单价** — 每行都写
3. **单位** — 公斤
4. **20匹以上规则** — 总匹数 × 0.2 = 总加重量，随机分配到各匹每匹+1kg
   - 例：340匹 × 0.2 = 68kg → 随机68匹各+1kg
5. **多页拆分** — 超过100匹自动分Sheet，每页最多100匹
6. **NO号** — 所有页都只写 `NO.{单号}`，不加"续N"后缀
7. **当前页/总页** — 每页底部填写（第1页/共4页...）
8. **金额** — 每页小计用公式，总计（本单应付）写死整单总金额

## 模板布局

### 大利标准模板（994×528）
- 10列×10行每页 = 100匹
- 重量列坐标 x=369...736 (每列~41px)
- 数据行 y=192...396 (每行~22px)

### 红呈双面板模板（898×454）
- 左右双面板，每面板6个重量列 = 每行12匹
- 约10行数据，通过水平投影自动检测行位置（y≈218-400区间）
- **逐行PSM 7 + Otsu二值化**（非PSM 6）：
  - 裁剪单行 → 3倍放大 → 灰度/归一化/sharpen → Otsu二值化 → PSM 7单行模式 eng数字
  - 第一行右侧末列值缺失时：从x=688区域逐格cellOCRWeight补充（正则去空格 `\d{2}\.\d`）
- **稀疏页回退**：行级OCR+半区补充仍不足50%总匹数时，整页PSM 6全量OCR提取
- 规格/品名通过全页chi_sim OCR提取（缺训练数据时标注"未识别"）

### 恒泽模板（907×528）
- 约2行数据，8个重量列
- 通过水平投影自动检测行位置（y≈140-190区间）
- 同红呈逐行PSM 7 + Otsu二值化方式提取重量值

## 本次更新记录（2026-07-09）

### 新功能
1. **合并单** — 同一客户多个产品合并为一张销售单，不同产品自动换行
2. **数据校验** — 图片总结行（总匹数+总公斤）必须与逐格提取一致才生成，否则拒绝
3. **催收文案** — 生成后自动复制「你好，麻烦结下货款{金额}」到剪贴板

### 新规则
4. **四面弹按米** — 单位=米(非公斤)，OCR不要用15-60kg过滤
5. **空96规则** — 部分供应商名义米数×0.96=实际米数，以实际计算金额
6. **小字OCR** — PSM 8/13直接读原始cell比PSM 7+Otsu更准

### Bug修复
7. **恒泽行检测** — 两条数据行被合并为一个70px宽行 → 需按投影网格线拆分
8. **多产品同行覆盖** — 产品2和产品1挤在同一行导致品名被覆盖 → 不同产品强制换行

### 关键解析链路
```
红呈双面板图片 → detectTemplate(898×454) → parseHongcheng
  → extractSummary (全页chi_sim → 总匹数/总公斤)
  → detectDataRows (水平投影 → 10行位置)
  → for each row: PSM 7 + Otsu 二值化提取 ≤12值
    → 第1行右侧末列缺失 → cellOCRWeight 逐格补充
  → 匹数<50%预期 → 整页PSM 6 全量OCR替代
  → applyKgCorrection (自动按比例修正到总结行公斤)
```
