# 大利纺织 销售单生成规则

## 使用命令

```
cd E:/projects/fabric-sales-agent
node scripts/pipeline.js "图片路径" --customer 客户名 --name 品名 --spec 规格 --no 单号 --color 颜色
```

### 单产品示例

```
node scripts/pipeline.js "图片.png" --customer 恒泽 --name 牛奶丝双磨 --spec 190CM180G --no 12 --color 白色
```

### 双产品示例（同品名不同颜色/价格）

```
node scripts/pipeline.js "图片.png" --customer 巫石 --name 牛奶丝双磨 --spec 185CM180G --no 13 --color 黑色 --price 12 --split 20 --color2 白色 --price2 11
```

> `--split N` = 前N匹为第一个颜色/价格，后面匹数为第二个
> `--price` 不传则自动从图片总结行计算单价

---

## 图片解析规则

1. **全页OCR** — 先4倍放大图片，用 `chi_sim` 语言读整页，提取总结行：
   - 总匹数（340匹）
   - 总公斤（10526.5公斤）
   - 总金额（111580.90元）
   - 单价 = 总金额 ÷ 总公斤（自动计算）

2. **逐格OCR** — 按固定坐标切每个格子（400×160px），Otsu二值化后用 `eng` 数字模式读

3. **噪音过滤** — 每行不足5个15-60kg值的直接丢弃（表尾空白行）

4. **总公斤修正** — 逐格之和如果与图片总结行不一致，按比例缩放到总结行公斤数

---

## 销售单规则

1. **品名/规格** — 只在每组第一行写，后续行留空（合并效果）
2. **颜色/单位/单价** — 每行都写
3. **单位** — 公斤
4. **20匹以上规则** — 总匹数 × 0.2 = 总加重量，随机分配到各匹每匹+1kg
   - 例：340匹 × 0.2 = 68kg → 随机68匹各+1kg
5. **多页拆分** — 超过100匹自动分Sheet，每页最多100匹
6. **NO号** — 所有页都只写 `NO.{单号}`，不加"续N"后缀
7. **当前页/总页** — 每页底部填写（第1页/共4页...）
8. **金额** — 每页小计用公式，总计（本单应付）写死整单总金额

## 输出文件

生成到 `data/msg/file/` 目录下：

| 文件 | 说明 |
|------|------|
| `大利纺织销售单-XFS{订单号}-NO{单号}.xlsx` | Excel销售单 |
| `大利纺织预览-XFS{订单号}-NO{单号}.html` | 网页预览 |

---

## 注意事项

- 图片需要是供应商送货单截图（994×528/页格式）
- 单号(`--no`)不传则自动取最大号+1
- 如果WPS打开过文件锁住，重新生成前先关掉WPS
